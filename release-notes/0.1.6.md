# DevNest 0.1.6

## Highlights

- Fixes Cloudflare Persistent Domain failures after switching local web servers between FrankenPHP and Apache in the same DevNest session.

## Added

- No new user-facing features in this patch release.

## Improved

- Fixed stale persistent tunnel state when stopping one web server and starting another from `Services`.
- Fixed `Go Live` / `Apply Live` incorrectly reusing old shared Cloudflare tunnel routes from the previous web server.
- Fixed false port-conflict errors such as `Stop the conflicting process before starting FrankenPHP` when the selected project is already configured for Apache, and the inverse Apache error after switching to FrankenPHP.
- Persistent tunnel actions now use DevNest's current in-session active tunnel routes instead of treating the previous `cloudflared` config file as the action source of truth.

## Notes

- Recommended for users using Cloudflare Persistent Domains with both Apache and FrankenPHP.
- After updating, restart DevNest once if a stale tunnel route is already active from the previous version.
