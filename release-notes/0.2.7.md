# DevNest 0.2.7

## Highlights

- Fixed stale service status on first app launch after Windows starts.
- Improved startup/workspace load time by moving port checks off the initial overview path.
- Fixed duplicate PHP service-state folders for full patch versions like `8.4.20`.
- DevNest now registers itself to launch when Windows starts.

## Improved

- Dashboard and Services now refresh after background boot restore finishes, with a single cold-start fallback if the boot event is missed.
- Workspace overview now loads service/project data first, then refreshes port conflict health asynchronously.
- Service and port status checks now use batched process lookup to reduce repeated helper calls.
- PHP runtime config, CLI, and FastCGI now share family-based managed config paths such as `service-state/php/8.4/php.ini`.
- Launch-at-login registration is best-effort and does not block the app if Windows denies the registry update.

## Verified

- Passed `npm run verify:fast`, frontend build, and focused Rust tests for process lookup, PHP config paths, and FastCGI config.

## Notes

- No database migration or UI workflow changes in this release.
- Existing full-version PHP state folders are legacy generated data and can be removed after backing them up.
