# DevNest 0.1.4

## Highlights

- Open-source polish release focused on cleanup, release flow hardening, and public-facing docs.

## Added

- Public repo docs for contributors and maintainers: `README.md`, `CONTRIBUTING.md`, `SECURITY.md`, `.env.example`, and `BEFORE_CODING.md`.
- Standalone webdocs build setup inside `src-webdocs` for easier static-site deployment.
- Optional `release:windows:all` flow to publish both NSIS and MSI installers alongside the default Windows release assets.

## Improved

- Cleaned release defaults so the updater and release scripts target the main public repository flow instead of the old separate release repo setup.
- Hardened `release:windows` so forks and local clones can build safely without accidentally publishing to the official update feed.
- Improved landing page, legal pages, metadata, and SEO setup for `devnest.one`.
- Cleaned public repo wording and licensing so the project is easier to understand as an open-source release.
- Fixed hardcoded release endpoint references in app-side fallback/update metadata wiring.

## Notes

- This release is mostly polish and infrastructure work rather than a large feature drop.
- Release maintainers should set `DEVNEST_GITHUB_REPO=hohphu8/devnest` in the current PowerShell session before running `npm run release:windows` or `npm run release:windows:all`.
