# DevNest 0.2.1

## Highlights

- Improved FrankenPHP Laravel Octane Worker reliability, observability, and app restart behavior.
- Added a cleaner Project Runtime experience with stable loading states and less scroll/layout jank.
- Hardened worker restore so DevNest no longer trusts stale Windows PIDs after reboot.

## Improved

- Octane health now reports worker uptime, PID, worker/admin ports, last error, log tail, and metrics when available.
- Uptime no longer resets on every health refresh while the same worker PID is still running.
- Request metrics parsing now supports broader FrankenPHP/Caddy request counter names and degrades cleanly when counters are unavailable.
- Laravel Octane workers can auto-restart after detected Laravel code/config/env changes.
- Boot restore now validates that the stored PID owns both managed worker/admin ports before treating a worker as running.
- Worker startup now waits longer for cold Laravel/FrankenPHP boots before reporting a port-open failure.
- Runtime and Provisioning tabs now use full-tab loading gates to avoid scroll jumps while initial data loads.
- Settings Web Servers now distinguishes FrankenPHP builds by embedded PHP family.

## Notes

- FrankenPHP Classic remains unchanged and is still the default.
- Laravel Octane Worker mode remains Laravel-only.
- PHP extension management stays in Settings PHP Tools; Project Runtime focuses on HTTP worker health.
