# DevNest 0.1.5

## Highlights

- Adds experimental FrankenPHP support for Windows, including managed downloads, project selection, service control, and embedded PHP compatibility handling.
- Fixes persistent Cloudflare domain failures for users upgrading from older app data.

## Added

- `FrankenPHP (Experimental)` is now available as a web server option for projects.
- FrankenPHP can be installed from `Settings -> Web Server Downloads`.
- FrankenPHP appears in Installed Runtimes with PHP Tools support for embedded PHP extensions and guarded functions.
- FrankenPHP projects now use the embedded PHP family from the active FrankenPHP runtime, so a separate standalone PHP install is not required.
- Added managed FrankenPHP PHP config and extension overlay support, including Redis package selection for thread-safe FrankenPHP builds.

## Improved

- Improved Add Project and Project Inspector behavior so selecting FrankenPHP auto-syncs the project PHP version to the active embedded PHP family.
- Fixed FrankenPHP embedded PHP version detection on Windows.
- Fixed FrankenPHP service startup with managed `php.ini`, `PHPRC`, and extension scan paths.
- Fixed FrankenPHP Redis extension install flow by separating thread-safe and non-thread-safe PHP extension packages.
- Fixed stale Apache/Nginx config cleanup when switching a project to FrankenPHP.
- Fixed persistent domain publishing when older DevNest data had project child tables still pointing at `projects_legacy`.
- Improved SQLite error wording so database operation failures no longer appear as misleading bootstrap failures.

## Notes

- Existing users should restart DevNest once after updating so the app can run the metadata repair migration automatically.
- FrankenPHP support is still experimental and currently targets classic mode, not worker mode or Laravel Octane UX.
- FrankenPHP carries its own embedded PHP runtime. Install standalone PHP only for Apache/Nginx projects or CLI usage.
- Apps that need MySQL, Redis, or extra PHP extensions still need those services/extensions installed separately for the selected runtime.
