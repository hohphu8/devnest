Privacy Policy

This Privacy Policy explains how DevNest handles information when you visit this site or use the DevNest desktop application. DevNest is built as a Windows-first local PHP workspace, so the normal product flow is local-first rather than account-first.

1. Website usage

This website is a static product and documentation site. DevNest does not intentionally collect personal information here through user accounts, private dashboards, or built-in contact forms.

Like any website, your browser, network provider, CDN, or hosting provider may still process standard request data such as IP address, browser version, request path, referrer, and timestamps in order to deliver pages.

2. Local application data

DevNest is designed to work primarily on your own Windows machine. Project metadata, runtime state, local domains, diagnostics, task history, snapshot history, and similar workspace data are generally intended to stay on your device so the app can provide local development features.

  • Tracked project metadata may include local paths, project names, framework hints, domains, runtimes, and related workspace state.
  • Runtime and database features may create local logs, backups, snapshots, task history, or rollback records on your machine.
  • Core local use of DevNest does not depend on a hosted DevNest account.

3. Network activity and third-party services

Some DevNest workflows may contact third-party services only when you choose to use features that require them. Examples include checking GitHub Releases for updates, downloading optional tools, or using Cloudflare tunnel-related sharing flows.

When those services are involved, their own privacy policies and terms apply in addition to this page. DevNest does not control or override third-party policies.

4. Telemetry and analytics

DevNest is not presented as an ad-supported or data-broker product. If future releases add optional telemetry, crash reporting, analytics, newsletter flows, or support intake forms, this policy should be updated before those features are treated as part of the public product.

5. Data control and retention

Because DevNest is local-first, you generally control the data the app creates on your own device. You are responsible for reviewing, deleting, backing up, exporting, or migrating local DevNest data when needed.

6. Changes to this policy

This Privacy Policy may be updated as DevNest evolves, especially if future releases introduce online accounts, cloud syncing, analytics, or support channels that process personal data more directly.

7. Contact

If you publish DevNest publicly, replace placeholder repository and support references with your real contact, release, and support channels.
