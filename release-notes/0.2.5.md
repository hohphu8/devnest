# DevNest 0.2.5

## Highlights

- Faster app startup with background restore for services and workers.
- Smoother Logs page performance for large log files.
- Lighter workspace refresh behavior after the initial overview load.

## Improved

- Fixed Nginx startup for longer local domains by increasing the generated server-name hash settings.
- Boot now keeps only required setup work blocking, then restores services, workers, FrankenPHP workers, and scheduled tasks in the background.
- Scheduled tasks keep the same 1-second cadence but reuse the scheduler database connection instead of reopening SQLite every tick.
- Log tail reads now load only the needed tail chunks while preserving the same log payload and line numbers.
- Workspace overview hydration now avoids extra immediate fetches on Projects, Services, Logs, Workers, and Tasks views.
- Added optional internal timing logs with `DEVNEST_PERF_LOG=1` for boot and workspace overview profiling.

## Verified

- Passed fast verification, Rust check, focused log reader tests, and manual QA for boot, tray, background restore, logs, scheduled tasks, and workspace hydration.

## Notes

- No database migration, command contract, or UI workflow changes in this release.
